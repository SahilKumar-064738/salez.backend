## Authentication

### POST /auth/register

Creates a new tenant account and returns JWT tokens immediately (auto-login).

**Request**
```json
{
  "businessName": "Acme Corp",
  "email": "admin@acme.com",
  "password": "securePass123",
  "displayName": "John Doe"
}
```

**Response — 201**
```json
{
  "success": true,
  "data": {
    "userId": "uuid",
    "tenantId": 42,
    "tenantSlug": "acme-corp",
    "email": "admin@acme.com",
    "accessToken": "eyJ...",
    "refreshToken": "eyJ...",
    "expiresAt": 1700000000
  },
  "message": "Account created successfully"
}
```

**Errors**

| Status | Code | Reason |
|--------|------|--------|
| 409 | `CONFLICT` | Email already registered |
| 422 | `VALIDATION_ERROR` | Missing/invalid fields |

**Axios**
```ts
const { data } = await axios.post('/auth/register', {
  businessName: 'Acme Corp',
  email: 'admin@acme.com',
  password: 'securePass123',
  displayName: 'John Doe',
});
const { accessToken, refreshToken, tenantId } = data.data;
```

## Contacts

All endpoints require `Authorization: Bearer <token>`.

### GET /contacts

Cursor-paginated contact list. Soft-deleted contacts excluded.

**Query Parameters**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `limit` | number | 50 | 1–100 |
| `cursor` | string | — | Pagination cursor from previous response |
| `stage` | string | — | `new` \| `contacted` \| `qualified` \| `converted` \| `lost` |
| `search` | string | — | Searches name and phone (case-insensitive) |
| `tag` | string | — | Filter by tag |

**Response — 200**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "tenant_id": 42,
      "phone": "+919876543210",
      "name": "Rahul Sharma",
      "email": "rahul@example.com",
      "stage": "new",
      "notes": null,
      "tags": ["vip", "lead"],
      "created_at": "2024-01-01T00:00:00Z",
      "updated_at": "2024-01-01T00:00:00Z",
      "last_active": null,
      "deleted_at": null
    }
  ],
  "pagination": { "nextCursor": "eyJ...", "hasMore": true }
}
```

**Axios**
```ts
const { data } = await api.get('/contacts', { params: { limit: 50, stage: 'new' } });
const contacts = data.data;
const nextCursor = data.pagination.nextCursor;
```

### POST /contacts/bulk

Batch create up to 500 contacts. Checks plan limit before inserting.

**Request**
```json
{
  "contacts": [
    { "phone": "+919876543210", "name": "Rahul" },
    { "phone": "+919876543211", "name": "Priya" }
  ]
}
```

**Response — 201**
```json
{ "success": true, "data": [{...}, {...}], "message": "2 contacts created" }
```

**Errors**

| Status | Code | Reason |
|--------|------|--------|
| 429 | `LIMIT_EXCEEDED` | Batch would exceed plan contact limit |
| 422 | `VALIDATION_ERROR` | Invalid contact data or > 500 contacts |

## Error Reference

All error responses follow this shape:

```json
{
  "success": false,
  "error": "Human readable message",
  "code": "ERROR_CODE",
  "statusCode": 400
}
```

**Validation errors (422) also include:**
```json
{
  "details": [
    { "path": "email", "message": "Invalid email" }
  ]
}
```

### Error Code Reference

| HTTP | Code | When |
|------|------|------|
| 400 | `MISSING_FIELD` | Required field absent |
| 400 | `INVALID_PARAM` | Malformed URL param (e.g. non-numeric ID) |
| 401 | `UNAUTHORIZED` | Missing or invalid JWT |
| 401 | `INVALID_CREDENTIALS` | Wrong email/password |
| 401 | `ACCOUNT_DEACTIVATED` | User account disabled |
| 401 | `INVALID_REFRESH_TOKEN` | Expired or invalid refresh token |
| 403 | `FORBIDDEN` | Insufficient role |
| 403 | `FEATURE_NOT_AVAILABLE` | Plan feature not enabled |
| 404 | `NOT_FOUND` | Resource not found |
| 409 | `CONFLICT` | Email already registered |
| 409 | `DUPLICATE_ENTRY` | Unique constraint (e.g. duplicate phone) |
| 422 | `VALIDATION_ERROR` | Zod schema validation failed |
| 422 | `INVALID_REFERENCE` | FK constraint violation |
| 429 | `LIMIT_EXCEEDED` | Plan resource limit reached |
| 500 | `INTERNAL_ERROR` | Unexpected server error |
